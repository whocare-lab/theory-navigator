# 博士論文理論導航｜力成科技潛伏型資本積累策略研究

> **一個為 DBA 論文打造的理論檢索工具**  
> 將管理理論與企業個案的轉型軌跡一一對應，讓文獻回顧不再大海撈針。

---

## 這是什麼？

這個倉庫收錄兩份導航文件，協助撰寫以**力成科技（6239）FOPLP 先進封裝轉型**為個案的 DBA 論文：

| 文件 | 用途 | 適合誰 |
|------|------|--------|
| **[`index.md`](./index.md)** | 從《管理與組織研究常用的60個理論》中篩選出與「策略轉型、資本積累、生態系重組」相關的理論，標註高/中/低相關性 | 需要快速判斷「哪些理論值得讀」的論文作者 |
| **[`theory-stage-mapping.md`](./theory-stage-mapping.md)** | 將篩選出的理論對應到力成科技 2010–2026 年的四階段轉型軌跡（輝煌→崩盤→潛伏→躍遷） | 需要「把理論說進個案時間軸」的論文作者 |

---

## 四階段轉型架構

本論文以蘇軾《定風波》的意象，描述力成科技的轉型歷程：

```
第一階段（≤ 2010）  騎馬奔馳  →  顯性資本的輝煌盛世
第二階段（2011–2015）墜馬求生  →  爾必達破產引發的戰略收縮
第三階段（2016–2025）竹杖芒鞋  →  潛伏型資本積累的十年深蹲
第四階段（2026–）    輕勝馬    →  AMD EFB 驗證完成的生態位躍遷
```

## 如何使用

### 情境一：寫文獻回顧（Ch2）時
1. 打開 [`index.md`](./index.md)
2. 先看「本論文已採用的核心理論」確認骨架不動搖
3. 再從「高相關 11 個理論」中挑選 3–5 個，對照「論文可能位置」欄位寫入 Ch2

### 情境二：寫 Findings（Ch4）時
1. 打開 [`theory-stage-mapping.md`](./theory-stage-mapping.md)
2. 找到你正在分析的階段（如第三階段「竹杖芒鞋」）
3. 直接對照「對應機制」與「論文應用」欄位，將理論嵌入個案敘事

### 情境三：口試準備時
1. 打開 [`theory-stage-mapping.md`](./theory-stage-mapping.md) 最後的「理論使用總表」
2. 這張矩陣就是「哪個理論用在哪個階段」的一頁紙備忘
